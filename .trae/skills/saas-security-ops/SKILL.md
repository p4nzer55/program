## 一、云监测接入流程

### 1.1 接入前准备
- 需要客户填写《云监测网站环境调查表》
- 确认扫描授权书（若需要）
- 收集备案 IP 信息

### 1.2 接入流程
1. **申请阶段**：客户提交云监测服务接入申请
2. **配置阶段**：
   - 在管理端添加监测站点
   - 配置监测参数
   - 确认监测合同
3. **验证阶段**：确认监测正常运行

### 1.3 功能操作
- 导出监测自定义汇总报告（日报、月报、年报）
- 导出任意时间单站点监测报告
- 导出实时报告
- 制作脱敏报告模板（去除 IP、单位、域名等敏感信息）
- 查看可视化大屏
- 平台登录密码修改/重置
- 安全事件告警查看
- 通过 URL/用户手机号码查询合同

## 二、玄武盾防护管理

### 2.1 防护接入流程
1. 客户提交《云防护环境调查表》
2. 配置 DNS 解析（接入玄武盾）
3. 在平台添加防护站点
4. 配置防护规则

### 2.2 防护模式
- **防护模式**：开启全量防护
- **检测模式**：仅记录日志不拦截
- **转发模式**：仅转发流量不防护

### 2.3 核心功能操作

#### 一键关停
- 开启/关闭一键关停功能
- 观察状态及生效时间

#### 永久在线
- 需要先开启学习模式，学习 3 天后创建镜像
- 开启永久在线功能
- 分析站点状态

#### 回源/回切
- **回源站点**：通过 DNS 完全绕过玄武盾
- **回切站点**：通过 DNS 接入玄武盾

#### 误拦截处理
当出现 403 拦截时：
1. 确认正常业务被误拦截
2. 在 URL 级别或站点级别解封该防护策略
3. 验证恢复正常访问
4. 删除策略限制，恢复正常拦截功能

#### 端口配置
- 访问未添加的端口或配置未生效会报 **422 错误**
- 解决方法：加入该端口或联系首席重新下发配置

#### 防护功能
- 防扫描
- 防 CC
- 访问控制
- 区域访问控制

#### 日志与报表
- 查看、下载访问日志
- 通过域名等信息查询合同
- 导出防护日报、月报
- 制作脱敏模板

## 四、问题排查手册

### 4.1 403 拦截排查
1. 确认拦截时间点
2. 查看访问日志
3. 检查防护规则
4. 如为误拦截，按误拦截处理流程操作

### 4.2 422 错误排查
422 错误表示访问了未添加的端口或端口配置未生效
- 检查站点配置的端口列表
- 确认端口是否已添加
- 如未添加，添加该端口或联系首席重新下发配置

### 4.3 5XX 错误排查
参考《玄武盾 5XX 响应码及常见快速排障措施》文档

常见 5XX 错误原因：
- 源站服务器故障
- 网络连接问题
- 配置错误

### 4.4 站点无法访问排查
1. 检查 DNS 解析是否正确
2. 确认防护模式设置
3. 查看访问日志
4. 检查源站状态
5. 如需临时回源，可开启回源站点功能
